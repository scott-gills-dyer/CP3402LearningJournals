## Content Insights
Using Docker introduced the concept of environment separation, where the application and database run in isolated containers rather than directly on the host system. This highlights how development environments can be standardised and replicated, reducing issues caused by differences between local setups and production systems.

Working with a local WordPress and MariaDB setup also showed how multiple components interact within a system. The CMS is not operating independently, but relies on a database connection and configured environment to function correctly. This reinforces that web applications are composed of interdependent services rather than a single standalone system.

In addition, the use of Docker changes how deployment is approached. Instead of focusing only on code changes, the environment itself becomes part of the system that needs to be managed and configured. This shifts development from simply building features to also considering how those features will run consistently across different environments.

## Career / Employability / Learning Insights
This week highlights the importance of controlled environments when working with systems. Using Docker to run a local WordPress and database setup reflects how environments are separated and managed to ensure consistent behaviour across development and production systems. This is directly relevant to infrastructure and systems roles, where environment consistency is critical to avoiding deployment and configuration issues.

It also demonstrates that development is not limited to writing code, but includes managing the environment that the system runs in. Tools like Docker introduce concepts such as containerisation and reproducibility, which are commonly used in modern infrastructure, deployment pipelines, and cloud-based systems.

Additionally, this reinforces the need to understand how different system components interact. The CMS, database, and environment configuration must all function together correctly, which reflects how real-world systems are built from multiple dependent services rather than a single application.
