## Content Insights
Learning to use Docker provided an introduction to early stage website development and testing, rather than going straight into live builds or GitHub deployment. I learned that using a local environment allows changes to be tested safely before being exported to production.

I am also feeling more confident in navigating WordPress, as I now have a better understanding of where settings, content blocks, and template options are located within the CMS. While I still needed to refer to documentation for some template-related issues, I found that WordPress includes more built-in functionality than I previously thought. Instead of needing to import external frameworks like Bootstrap (as I did in a previous practical), there are already built-in features for buttons and styling that can be customised through the CMS.

This changed how I understood development a bit, because before this I  assumed most website updates were just tested then deployed on GitHub or Host without needing to run everything locally first. Seeing how Docker lets the CMS and database run together on my own machine helped me understand how full environments can be tested before anything is deployed.

## Career / Employability / Learning Insights
I learned that web developers generally do not update websites directly on live production environments, and instead make use of local development tools and staging sites to test features before implementation. This acts as an important buffer for bug finding, as well as peer or client review before deployment.
Becoming more familiar with WordPress and learning how to use proper local development tools such as Docker is relevant to how development and deployment workflows are handled in professional environments. It reflects the expectation that developers understand how to safely test and manage changes before they are pushed to production systems.
